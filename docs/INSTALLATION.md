# 📦 Installation Guide

<p align="center">
  <strong>English</strong> · <a href="./INSTALLATION.zh-CN.md">简体中文</a>
</p>

Animal Island is a standard Hexo theme. Install it into any Hexo site's `themes/` directory by cloning from GitHub, copying the source folder, or using a local symlink.

> ⚠️ **Usage restriction:** This project is for personal learning, research, and non-commercial demonstration only. Commercial use is prohibited. Read [LEGAL_NOTICE.md](../LEGAL_NOTICE.md) before using it.

## 🌿 Option 1: Clone from GitHub

Recommended for most users.

```bash
cd /path/to/your-hexo-site
mkdir -p themes
git clone https://github.com/timoforge/hexo-theme-animal-island.git themes/animal-island
```

Enable the theme in the Hexo site root `_config.yml`:

```yaml
theme: animal-island
```

## 🧺 Option 2: Copy downloaded source

Use this when installing from a release archive or another local source folder.

```bash
cd /path/to/your-hexo-site
mkdir -p themes
cp -a /path/to/hexo-theme-animal-island themes/animal-island
```

## 🔗 Option 3: Symlink for local development

Use this when maintaining the theme locally and testing it in one or more Hexo sites.

```bash
cd /path/to/your-hexo-site
mkdir -p themes
ln -s /path/to/hexo-theme-animal-island themes/animal-island
```

Changes to `/path/to/hexo-theme-animal-island` will affect every Hexo site using this symlink.

## ⚙️ Add theme override config

Copy the example override config to the Hexo site root.

```bash
cp themes/animal-island/examples/_config.animal-island.yml _config.animal-island.yml
```

Hexo merges the theme default `_config.yml` with the site-level `_config.animal-island.yml`; the site-level file has higher priority. Keep personal settings in the site root so theme upgrades do not overwrite them.

## 🗺️ Recommended pages

The default menu includes search, categories, tags, archives, and about. Archives are generated by Hexo, but the other pages should be created manually.

```bash
hexo new page search
hexo new page categories
hexo new page tags
hexo new page about
```

Set `type` in each page's front matter.

```yaml
# source/search/index.md
---
title: Search
type: search
---
```

```yaml
# source/categories/index.md
---
title: Categories
type: categories
---
```

```yaml
# source/tags/index.md
---
title: Tags
type: tags
---
```

## 🧪 Build verification

Run Hexo generation from the Hexo site root.

```bash
hexo clean && hexo generate
```

If the site already defines its own build script, you can use that instead.

```bash
npm run build
```

## 📁 Theme folder name rule

The Hexo `theme:` value must match the folder name under `themes/`.

Recommended:

```text
themes/animal-island
```

```yaml
theme: animal-island
```

If you keep the folder name `hexo-theme-animal-island`, configure Hexo like this:

```yaml
theme: hexo-theme-animal-island
```

The override config file name must match too:

```text
_config.hexo-theme-animal-island.yml
```
